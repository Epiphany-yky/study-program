# TCP Echo服务器设计文档

## 整体架构
单线程阻塞式TCP服务器，核心流程：
1.  socket创建 → 端口绑定 → 监听 → 阻塞等待连接
2.  接受连接 → 读取客户端数据 → 原封不动回写 → 关闭连接
3.  循环等待下一个连接

## 核心模块说明
### 1. 网络通信模块
- 系统调用：socket、bind、listen、accept、read、write
- 端口复用：通过setsockopt(SO_REUSEADDR)解决端口占用问题
- 优雅退出：通过pselect超时轮询+SIGINT信号处理，解决accept阻塞卡死问题

### 2. 错误处理模块
- 全流程错误捕获：socket/绑定/监听/读写失败均有提示
- 资源释放：失败时自动关闭socket，避免资源泄漏

## 后续优化方向
- 多线程改造：通过线程池实现高并发
- HTTP协议解析：支持GET请求、静态资源返回
- 日志系统：支持日志分级、文件输出
